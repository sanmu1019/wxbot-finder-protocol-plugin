# 故障排查

## 插件没有响应

检查：

1. `enabled=true`。
2. wxbot 已重启。
3. 引用了视频号卡片。
4. 发送了 `协议解析` 或配置中的触发词。
5. 群聊或私聊没有被禁用。

直接转发卡片默认不触发。

## 缺少 objectId 或 objectNonceId

机器人消息上下文必须保留完整引用卡片 XML。插件读取：

```text
content
original_content
outer_content
```

如果框架只保留卡片标题或摘要，协议直刷无法工作。

## current bot wxid is unavailable

检查：

```text
Config.CURRENT_BOT_WXID
Config.EXPECTED_BOT_WXID
get_current_login_wxid()
```

至少一个需要返回当前登录 bot 的 wxid。

## login data not found

protocol-core 没有找到该 wxid 对应的登录态。

检查：

- bot 是否已登录。
- protocol-core 与 Redis 是否使用同一套数据。
- wxid 是否匹配。
- 登录态是否过期。

## protocol response has no downloadable video

接口成功但 `media` 为空，或者媒体 URL 缺少必要参数。

检查 protocol-core 是否成功提取：

```text
field 12 -> decode_key
field 13 -> url_token
```

## 下载失败

常见原因：

- URL token 已过期。
- CDN 网络不可达。
- 文件超过 `video_max_size_mb`。
- 下载超时。
- URL 被截断。

协议返回后应立即下载，不要长时间缓存媒体 URL。

## 下载后不是视频

检查：

- 是否返回 `decode_key`。
- decodeKey 是否为十进制数字。
- Key 与 URL 是否来自同次响应。
- protocol-core unknown field 解析是否正确。

## 视频发送失败

插件会自动降级为文件发送。两者都失败时检查：

- bot 文件上传接口。
- `WECHAT_HTTP_PORT`。
- 文件体积。
- 临时目录权限。
- wxbot 日志。

## 与短链插件重复回复

两个插件同时启用且触发词相同会重复处理。

建议：

```text
短链插件：解析
协议插件：协议解析
```

或者只启用其中一个。

## 日志脱敏

可以记录：

```text
objectId
nonce 长度
媒体数量
URL 长度
查询参数名称
是否有 decodeKey
```

不要记录：

```text
完整 nonce
完整 URL
decodeKey
登录态
Cookie
SessionKey
```
