# 安装与配置

## 适用环境

目标 wxbot 应包含：

```text
wxbot/
├── config/
├── core/
├── plugins/
└── utils/
```

同时存在可用的 protocol-core，并且已经为当前 bot 微信号建立登录态。

## 安装

```powershell
git clone YOUR_REPOSITORY_URL
cd wxbot-finder-protocol-plugin
python -m pip install -r .\requirements.txt
```

安装到 wxbot：

```powershell
.\install_local.ps1 -BotRoot "E:\path\to\your-bot"
```

首次安装会创建：

```text
wxbot/plugins/finder_protocol_parser/config.json
```

后续更新不会覆盖配置。

## 推荐配置

```json
{
  "enabled": true,
  "allow_group": true,
  "allow_private": true,
  "auto_detect": false,
  "allow_direct_share": false,
  "send_summary": true,
  "send_video": true,
  "send_failure_text": true,
  "video_max_size_mb": 300,
  "download_timeout": 45,
  "triggers": [
    "协议解析",
    "协议视频"
  ],
  "protocol_base": "http://127.0.0.1:9000/api",
  "protocol_timeout": 15,
  "protocol_cgi": 3906
}
```

## 触发模式

### 引用卡片

默认推荐：

```text
引用视频号卡片
-> 发送“协议解析”
```

该模式需要：

```json
{
  "auto_detect": false,
  "allow_direct_share": false
}
```

### 直接转发卡片

启用：

```json
{
  "allow_direct_share": true
}
```

直接转发即调用协议接口，可能造成误触发。建议只在私聊或受控群聊启用。

### 自动识别

```json
{
  "auto_detect": true
}
```

插件会处理上下文中检测到的视频号卡片。该模式调用频率更高，不建议默认开启。

## 与原短链插件共存

短链插件默认触发：

```text
解析
解析链接
视频号解析
```

协议插件默认触发：

```text
协议解析
协议视频
```

保留不同触发词即可共存。若只使用协议模式，可以禁用或删除原
`wxsph_parser`。

## protocol-core 地址

同机：

```json
{
  "protocol_base": "http://127.0.0.1:9000/api"
}
```

跨机器不应直接使用公网 HTTP。应通过 VPN、WireGuard、Tailscale 或受身份认证的
HTTPS 代理访问。

## 下载限制

```json
{
  "video_max_size_mb": 300,
  "download_timeout": 45
}
```

体积超过限制时，下载工具应中断并删除临时文件。

## 更新

```powershell
git pull
.\install_local.ps1 -BotRoot "E:\path\to\your-bot"
```

然后重启 wxbot。

## 卸载

停止 wxbot 后删除：

```text
wxbot/plugins/finder_protocol_parser/
```

删除前可单独备份 `config.json`，但不要上传包含实际网络地址或敏感配置的备份。
